# 要件定義書 — マンダラート デスクトップアプリ

## 概要

3×3グリッドで思考を階層的に展開する個人向けデスクトップアプリ。
各セルをクリックするとさらに3×3に展開できる「無限階層式」のマンダラートツール。
データはローカル SQLite に保存し、ログイン時に Supabase へ任意同期する。

---

## プラットフォーム

| 項目 | 内容 |
|------|------|
| 対象OS | macOS / Windows |
| フレームワーク | Tauri v2（Rust バックエンド + Vite + React フロントエンド）|
| 配布方式 | .dmg（macOS）/ .msi（Windows）直接配布 |
| データ保存 | ローカル SQLite（プライマリ）+ Supabase（任意同期）|

---

## グリッド仕様

- **階層式（無限展開）**: どのセルもシングルクリックで3×3に掘り下げ可能
- **並列展開**: 同じ階層に複数の3×3グリッドを作成できる
  - 画面には常に **1つのグリッドのみ** 表示
  - 並列グリッドが存在する場合のみ **左右ナビゲーションボタン**（← →）を表示
  - **「+ 新しいグリッドを追加」ボタン** で同じ階層に並列グリッドを作成
- 階層の深さ・並列数に制限なし

---

## 表示モード

| モード | 内容 |
|--------|------|
| 3×3 表示 | 現在の階層の1グリッドのみ表示。セル編集・階層移動のメイン画面 |
| 9×9 表示 | 現在のグリッド＋その直下サブグリッドを2階層まとめて表示 |

- 画面上部のトグルボタン `[3×3] [9×9]` でいつでも切り替え可能
- グリッドは画面サイズに応じて正方形を維持したまま最大化して表示

## セル操作

**セルが空のとき**
| 操作 | 動作 |
|------|------|
| シングルクリック | インライン編集を即開始 (textarea にフォーカス) |
| ダブルクリック | 無視 (すでに編集中扱い) |

**セルに入力があるとき**
| 操作 | 動作 |
|------|------|
| シングルクリック（周辺セル） | サブグリッドへ掘り下げ (無ければ新規作成) |
| シングルクリック（中央セル・ルートグリッド） | ダッシュボードへ移動 |
| シングルクリック（中央セル・サブグリッド） | 親グリッドへ戻る |
| ダブルクリック | インライン編集を開始 |

**共通**
| 操作 | 動作 |
|------|------|
| セル右上の `⋯` ボタン (hover で表示) | セル編集モーダルを開く (色 / 画像 / 長文) |
| Esc / 外側クリック (インライン編集中) | 保存して編集終了 |
| Tab キー | 現在のセルを保存して次のセルへ移動（IME変換中は無視） |
| Shift+Tab キー | 現在のセルを保存して前のセルへ移動 |

> シングルクリックとダブルクリックの判別は 220ms の待機で行うため、入力ありセルのドリルには約 220ms の遅延が発生する。空セルはドリルの選択肢がないので即座に編集モードに入る。

**セルのナンバリング（0 インデックス、DB の `cells.position` と一致）:**
```
┌───┬───┬───┐
│ 0 │ 1 │ 2 │
├───┼───┼───┤
│ 3 │ 4 │ 5 │   ← 4 = 中央
├───┼───┼───┤
│ 6 │ 7 │ 8 │
└───┴───┴───┘
```

**Tab 移動順:**
```
4 → 7 → 6 → 3 → 0 → 1 → 2 → 5 → 8 → 4（ループ）
Shift+Tab は逆順
中央セル（4）が空の場合は Tab で移動しない
```

> インポート時の周辺セルへの配置順も、この Tab 移動順（中央を除いた `7 → 6 → 3 → 0 → 1 → 2 → 5 → 8`）に従う。<br>
> 先頭のノードから順に、Tab を押して行き着く順に埋まっていく。

## セル入力バリデーション

- **中央セルが空のとき**: 周辺セルは入力不可（非活性表示）
- **周辺セルに1つでも入力があるとき**: 中央セルを空にできない
- 「空」の定義: `text` が空文字 **かつ** `image_path` が NULL

---

## チェックボックス (完了状態)

セルごとに「完了 / 未完了」を管理するチェックボックスを設ける。目標・タスク系のマンダラートで進捗を追えるようにする。

- **表示位置**: セル左上隅の小四角 (16×16)。チェック時は ✓ が表示される
- **表示 ON/OFF トグル**: エディタヘッダにトグルスイッチを配置。localStorage に永続化 (デフォルト OFF)
- **非表示条件**: トグル OFF / インライン編集中 / 9×9 small モードでは描画しない
- **セルとの干渉回避**: チェックボックス押下はセルクリック (ドリル) を発火させない (`stopPropagation`)
- **階層カスケード**:
  - **親チェック → 子全セルもチェック** (配下のサブツリー全体を done=1)
  - **子全セルが done → 親も自動 done** (ボトムアップ伝播)
  - **アンチェック → 親も連鎖 done=0** (invariant 維持)。子孫の done 状態はユーザーの管理下に置くため保持
- **永続化と同期**: `cells.done` (SQLite INTEGER 0/1、Supabase BOOLEAN) を push/pull/realtime で同期
- **Undo スタックには入れない**: チェックのトグルは単純操作として扱う

## セルのドラッグ＆ドロップ

| ドラッグ元 | ドロップ先 | 挙動 |
|-----------|-----------|------|
| 周辺セル | 周辺セル | サブツリーごと入れ替え（`swapCellSubtree`）|
| 中央セル | 入力ある周辺セル | 内容のみ入れ替え（`swapCellContent`）|
| 中央セル | 空の周辺セル | サブツリー全体をコピー（`copyCellSubtree`）|
| 入力ある周辺セル | 中央セル | 内容のみ入れ替え |
| 空の周辺セル | 中央セル | 何もしない |

## 階層間データ移動

### カット＆ペースト
- 右クリックのコンテキストメニューから カット / コピー を選択
- キーボードショートカット（⌘X / ⌘C / ⌘V）
- カット後は元セルがグレーアウト表示

### ストックエリア
- セルをストックへコピー → 別の階層で貼り付け
- ストックはアプリを閉じても永続保存（SQLite）
- エディタ右サイドパネルに配置

**セル → ストック**
- コンテキストメニュー「ストックに追加」または、セルをストックパネルのドロップゾーンへ D&D
- ドラッグ開始時にサイドパネルは自動的に「ストック」タブへ切り替わる
- **周辺セル**: そのセルの子グリッド群（サブツリー）を含めて保存
- **中央セル**: そのセルが所属するグリッド自体（8 周辺セル + その下のサブツリー）を含めて保存
  - 「中央セルの subtree = 自分がテーマとしているグリッド全体」という解釈

**ストック → セル**
- ストックアイテムをセルへドラッグしてドロップ
- **空のセルのみドロップ可能**（入れ替えは行わない）
  - ドラッグ中、空セルのみハイライト表示される
- ドロップ時、ストック保存時のセル内容 + サブツリー全体をターゲットに適用
- ストックアイテムは消費されない（何度でもペースト可能）

## マンダラート管理

- 複数枚作成可能
- **タイトル = ルート中心セルのテキスト**
  - 別途「ファイル名」を付ける運用は廃止
  - `mandalarts.title` カラムはルート中心セルのキャッシュとして自動同期 (`updateCell` で position=4 + root grid に書き込むと同時に更新)
  - ダッシュボードの表示・検索・ソートは `mandalarts.title` を参照する
- **ホームへの遷移フロー**:
  1. 全セルが空のままホームへ → マンダラートを自動削除してダッシュボードへ
  2. 内容が入力済み → そのままダッシュボードへ (タイトル入力ダイアログは無い)
- ダッシュボードで削除・複製が可能 (リネームはマンダラートを開いて中心セルを直接編集する形に統一)
- 複製ではタイトルにサフィックス (「〜 のコピー」等) を付けない。同じ名前のマンダラートが複数存在しても問題なし

### ダッシュボードカードの表示
- 各カードはルート中心セル 1 つを等倍で並べた「セルのグリッド」のような見た目
- アスペクト比は正方形固定、一定サイズ (行/列の数は画面幅に応じてレスポンシブ)
- 中身はルート中心セルのテキストを小さめフォントで表示。長文でも可能な限り見えるよう複数行 + line-clamp
- 更新日 / 複製 / 削除は hover 時のみ表示

### ゴミ箱
- ダッシュボード上部の「ゴミ箱」ボタンから `TrashDialog` を開くと、`deleted_at IS NOT NULL` のマンダラートが削除日時の新しい順に並ぶ
- 各行で「復元」「完全削除」が選べる
- **完全削除の 2 クリック確認**: Tauri v2 の WebView は `window.confirm` が無効なので、ブラウザダイアログは使わず「1 回目クリックでボタン表記を『本当に削除?』に切替、2 回目で実行、4 秒放置で自動リセット」という state ベース UI を採用
- **完全削除の cloud 反映**: ローカル SQLite から物理削除した後、サインイン中であれば Supabase 側も `cells → grids → mandalarts` の順で `delete` する。これをやらないと次回 pull で cloud の `deleted_at` 付き行が再挿入されてゴミ箱に復活してしまう

## 認証・クラウド同期

- ログイン不要で全機能使用可能 (ローカル専用モード)
- Supabase Auth によるサインインで「クラウド同期モード」になる
- **対応済み**: メール + パスワード / Google OAuth / GitHub OAuth
- **同期方式**:
  - 起動時・サインイン時に `syncAll` で pull → push の順にフル同期
  - セル編集時に `updated_at` / `synced_at` の差分で push を走らせる
  - Supabase Realtime (`postgres_changes`) で別デバイスの変更を購読 → ローカル DB に反映 → UI が自動更新
  - 競合解決: `updated_at` last-write-wins
- **削除の同期**: ソフトデリート (`deleted_at`) で実現。オフラインで削除してもオンライン復帰時にクラウドへ伝播する。別デバイスでの削除は realtime / pull で不可視化される
- 詳細は [`cloud-sync-setup.md`](./cloud-sync-setup.md) 参照

## 表示設定

### ダークモード

- Tailwind v4 の `@custom-variant dark` を使ったクラスベース実装
- `themeStore` が `light` / `dark` / `system` の 3 値を localStorage に保存
- ヘッダの `ThemeToggle` (`☀ ◐ ☾`) で切り替え。`system` の場合は OS の設定変更にも自動追従

### 文字サイズ

- ヘッダの `A− / 現在の % / A＋` ボタンで調整
- 内部的には `fontLevel` (-10 〜 +20 の整数) を保持し、実効倍率は `1.1^level` の乗算式
- 最小 level -10 → 約 39%、 level 0 → 100%、最大 level +20 → 約 673%
- localStorage に永続化

## Undo / Redo

- `⌘Z` / `⌘⇧Z` / `⌘Y` (Mac)、`Ctrl+Z` / `Ctrl+Shift+Z` / `Ctrl+Y` (Windows)
- 対象: テキスト編集 / 色変更 / D&D (SWAP_SUBTREE / SWAP_CONTENT / COPY_SUBTREE) / セル移動
- D&D の COPY_SUBTREE は「target の事前状態 + 新規作成された grid ID」を記録して undo でそれらを削除・復元する

## デザイン

- シンプル・ミニマル・白基調のクリーンデザイン、ダークモードあり
- Tailwind CSS v4 によるスタイリング
- 背景色: プリセットカラーのみ (フルカラーピッカーなし)

### フォント

- システムフォント (macOS: SF Pro + ヒラギノ角ゴ) をそのまま使用、独自フォントはバンドルしない
- デフォルトウェイトは `index.css` で **300 (Light)** に固定
- 中心セルは太字化せず周辺セルと同じウェイト。強調は外枠の太さで表現する
- 詳細は [`typography.md`](./typography.md)

### セルのビジュアル

- **3×3 表示**
  - 中心セル: `border-[6px] border-black` (黒 6px)
  - 周辺セル (子グリッドあり): `border-2 border-black` (黒 2px)
  - 周辺セル (子グリッドなし): `border border-gray-300` (極細グレー)
  - 「子グリッドあり」の判定は「子グリッドが存在する」ではなく「子グリッドの**周辺セルに入力がある**」。`fetchChildCountsFor` の SQL で `position != 4 AND (text != '' OR image_path IS NOT NULL)` を条件に `COUNT(DISTINCT grid_id)` を返している。ドリル直後の中心セル自動コピーだけの状態では border は細いまま
  - セル間の余白 (gap) なし。セル同士は隣接して並ぶ
  - 中心セルが空でも周辺セルの見た目は変化しない (以前のグレーアウト挙動は廃止)
- **9×9 表示**
  - 各サブグリッドは `gap-px bg-gray-300` を持つラッパー + 内側セルの `bg-white` で「セル同士が共有する 1 本の極細グレーの境界線」を描画
  - サブグリッドラッパーの外枠: 中央 = `border-[6px] black`、既存の非中央 = `border-2 black`、空 (子グリッドなし) = `border-2 gray-300`
  - サブグリッド内の中心セルだけ `border-2 border-black -m-px z-10` で gap-px を跨ぐ黒枠を描画、周辺セルは枠なし (フラット)
  - 中央ブロックはルートグリッド自体の 9 セルを表示する
- **画像の扱い**
  - セルに画像と文字が両方入っている場合、確定表示では **画像のみ** を表示 (`absolute inset-0 object-cover`、不透明)
  - テキストはインライン編集中 / 拡大エディタ中のみ見える
  - パンくず末尾ラベル / ダッシュボードカードも「テキスト優先、空なら画像」のフォールバック
- **テキスト配置**
  - 左寄せ・上詰め (`items-start justify-start` + `text-left`)
  - セル外縁からテキストまでの「見える余白」は 3×3 で **18px**、9×9 で **6px** に統一
  - セルごとに border 幅が違う (1〜6px) ので、[`Cell.tsx`](../src/components/editor/Cell.tsx) は `textInsetPx = targetPadPx - borderPx` を計算して `absolute` の inset をインラインスタイルで動的補正する
  - 長文は上から流れて下方向にオーバーフローする。セル全体の `overflow-hidden` でクリップ
  - 左右の視覚バランスは短文で崩れる可能性があるが、CJK 長文は `break-all` で各行が幅いっぱいまで埋まるので実用上問題ない
- **ダッシュボードカード**
  - 130×130 の正方形タイル + `border-2 border-black` (以前の青枠は撤去)
  - 内側 `p-3` (12px) + 左寄せ・上詰め (`items-start` + `text-left`)
  - テキストは `line-clamp-6` で 6 行まで、超過分は `…`
  - タイトルが空で画像があればカード全面に画像を表示
- **インラインエディタの拡大**
  - インライン編集中に textarea をダブルクリックすると、`position: fixed` でサブグリッド全体を覆う拡大エディタが開く
  - 下部ツールバーに色選択 (プリセット 10 色) + 画像アップロードを集約
  - ツールバーの `onMouseDown={e => e.preventDefault()}` で textarea の blur を抑止
  - 従来あった `⋯` 詳細編集モーダルは**廃止済み**

### 並列グリッド操作

- **追加ボタン `+`**: 3×3 / 9×9 グリッドの右側。末尾かつ中心セルに入力がある場合のみ表示。クリックで新しい並列グリッドを作成し、元グリッドの中心セルの内容を新グリッドの中心セルに自動コピー
- **戻るボタン `<`**: グリッド左側。`parallelIndex > 0` の場合のみ表示
- **進むボタン `>`**: グリッド右側。次の並列グリッドが存在する場合のみ表示 (末尾の場合は `+` に切替)
- **旧 UI 廃止**: 画面上部の `1/2` 表示と下部の「+ 新しいグリッドを追加」ボタンは撤去済み
- **空グリッドの自動削除**: 離脱時 (並列切替 / パンくず / ホーム) に中心セルが空のグリッドは `cleanupGridIfCenterEmpty` で soft-delete される。最後の子グリッドだった場合はドリル元の親セルも連動して空にする

### アニメーション

- **並列スライド**: `<` `>` `+` でグリッドが切り替わる際に横方向のスライド (320ms)
- **ドリル軌道 (Orbit)**: ドリルダウン / ドリルアップ / 初回表示で、クリックした要素の移動 + 他要素の時計回り段階表示 (約 1000〜1080ms)。3×3 表示ではセル単位、9×9 表示ではサブグリッドブロック単位で同じ演出を再生
- **表示モード切替 (View Switch)**: 3×3 ↔ 9×9 のトグルで連続性のある演出
  - `to-9x9` (約 1195ms): 現在の 3×3 が中央原点で 1/3 に縮小しつつ、周辺 8 ブロックが時計回りに fade-in。終端で実 9×9 中央ブロックへクロスフェードして swap pop を回避
  - `to-3x3` (約 1080ms): 中央ブロックの 9 セルが時計回り `[7,6,3,0,1,2,5,8,4]` (中心最後) で 3×3 natural 位置へ拡大展開、周辺 8 ブロックは fade-out
- 詳細は [`animations.md`](./animations.md) 参照
